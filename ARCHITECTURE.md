# GeoInsight BI 概要设计文档

## 1. 项目概述
**GeoInsight BI** 是一个现代化的地理信息商业智能平台。它采用“微内核+插件”的设计理念，允许用户在动态交互的地图底图上，根据需求自由挂载、排列和组合各种分析组件（如统计图表、AI 聊天助手、图层管理器等）。

### 1.1 设计目标
*   **高度模块化**：功能通过插件实现，核心框架与业务逻辑解耦。
*   **动态交互**：支持插件窗口的自由拖拽、缩放和布局管理。
*   **AI 驱动**：集成 Google Gemini 模型，提供地理空间数据的智能分析与问答。
*   **高性能地图**：基于 OpenLayers 构建稳定、流畅的地图底座。

---

## 2. 系统架构
系统采用分层架构设计，确保了核心引擎的稳定性和外围功能的灵活性。

### 2.1 架构层次
1.  **基础设施层 (Infrastructure)**：包括 React 19 运行时、Vite 构建工具、Tailwind CSS 样式引擎。
2.  **核心内核层 (Kernel - MapCore)**：负责 OpenLayers 地图实例的初始化、投影转换、全局坐标同步及底图管理。
3.  **引擎层 (Engine - LayoutEngine)**：负责插件窗口的生命周期管理、层级控制（Z-Index）、拖拽缩放逻辑及持久化布局。
4.  **插件层 (Plugins)**：具体的业务功能单元，如 `ChartPlugin` (图表)、`ChatPlugin` (AI)、`ListPlugin` (数据列表) 等。
5.  **状态管理层 (State - Zustand)**：统一管理主题、插件实例状态、地图视图状态等全局数据。

---

## 3. 核心模块设计

### 3.1 插件系统 (Plugin System)
*   **PluginRegistry**：插件注册表，定义了插件的元数据（名称、图标、组件类型）。
*   **PluginInstance**：插件实例，存储在全局 Store 中，包含唯一的 ID、当前位置 (x, y)、尺寸 (w, h) 及自定义配置。
*   **动态加载**：通过 `registerPlugins()` 在应用启动时自动扫描并注册所有可用模块。

### 3.2 布局引擎 (Layout Engine)
*   采用绝对定位 + 响应式容器设计。
*   支持“窗口化”体验：每个插件运行在一个独立的容器中，具备标题栏、关闭按钮和交互反馈。

### 3.3 地图核心 (Map Core)
*   通过 `MapCoreProvider` 提供 Context，使任何插件都能轻松获取地图实例并进行交互（如缩放至特定区域、添加标记）。

---

## 4. 关键技术栈
| 维度 | 技术选型 | 说明 |
| :--- | :--- | :--- |
| **前端框架** | React 19 | 利用最新的并发渲染特性和 Hook 体系 |
| **地图引擎** | OpenLayers (ol) | 强大的开源 GIS 库，支持多种投影和大数据量渲染 |
| **状态管理** | Zustand | 轻量级、高性能的状态管理，适合频繁更新的布局数据 |
| **AI 模型** | Google Gemini API | 提供自然语言处理与数据洞察能力 |
| **可视化** | Recharts | 基于 SVG 的响应式图表库 |
| **样式** | Tailwind CSS | 实用优先的 CSS 框架，支持动态主题切换 |

---

## 5. 数据流向设计
1.  **用户交互**：用户点击“添加插件”按钮。
2.  **状态变更**：Zustand Store 更新 `plugins` 数组，生成新的插件配置。
3.  **组件渲染**：`LayoutEngine` 监听 Store 变化，动态映射并渲染对应的插件组件。
4.  **跨插件通信**：插件 A 修改 Store 中的共享数据（或操作 MapCore 实例），插件 B 响应式更新（例如：点击列表项，地图自动跳转位置，图表更新数据）。

---

## 6. UI/UX 设计规范
*   **主题系统**：支持 Light / Dark 模式，通过 Tailwind 的 `dark` 类名进行全局切换。
*   **视觉风格**：采用玻璃拟态（Glassmorphism）和简洁的 Slate 色系，强调专业感与科技感。
*   **响应式**：虽然以大屏 BI 为主，但布局引擎具备基础的自适应能力。

---

## 7. 扩展性说明
*   **新插件开发**：只需在 `plugins/` 目录下创建新组件，并在 `plugins/index.ts` 中注册，即可在 UI 中直接使用。
*   **数据源扩展**：`services/api.ts` 封装了统一的数据请求接口，易于对接不同的后端 GIS 服务。
